## 🃏 5. ПРОДУКТОВА КАРТА (ProductCard)

**Визия:** Абсолютно същата карта като в секцията "Топ продукти" на главната страница, но мащабирана надолу за 4-колонна решетка.

### Справка за изходния дизайн (ProductsSection.tsx):
- Контейнер: `rounded-[2rem]`, `shadow-sm hover:shadow-xl`, бял фон, `border`
- Снимка: `h-56`, `object-contain p-4`, `group-hover:scale-105` (zoom)
- Горен градиент: `from-black/40 via-transparent` над снимката
- Бейджове (горе): Категория (цветен pill) + Енергиен клас (зелен pill)
- Марка: `text-xs font-bold text-[#00B4D8] uppercase tracking-wider`
- Модел: `text-[1.35rem] font-bold text-gray-900`
- Спецификации: ⚡ kW, ❄️ Охл/Отопл, 🔄 Инвертор – с цветни икони
- Extras: Pill чипове с ✓ в зелено на сив фон (`bg-gray-50 border border-gray-100`)
- Рейтинг: Жълти звезди + число + брой отзиви
- Цена: `text-3xl font-extrabold` + `с монтаж от X лв` в сиво
- Бутон: `bg-[#EBF5FF] text-[#00B4D8] hover:bg-[#00B4D8] hover:text-white rounded-full`
- Текст на бутон: **"Направи запитване"**
- Дъно: Линк "Виж пълни характеристики →" (сив, `hover:text-[#FF4D00]`)

## 🚀 11. PREMIUM UX – СЛЕДВАЩО НИВО

Вдъхновено от Apple, Dyson, и Daikin.com – функции, които правят разликата между "хубав сайт" и "незабравимо изживяване".

### 🧭 A. "Помогни ми да избера" – Guided Buying Wizard
Wizard с 4 въпроса (тип помещение → м² → приоритети → бюджет) → показва ТОП 3 препоръки. Анимиран progress bar, slide transitions между стъпките.

### 🔮 A2. Parallax Hero Section
При скрол, елементите в CatalogHero се движат с различни скорости:
- Текст: 0.5x скорост
- Фонови сфери: 0.3x скорост  
- Stats banner: 0.8x скорост
- Създава дълбочина и premium усещане

### ✨ A3. Text Scramble Effect
При hover на заглавията (H1, H2), буквите се "разбъркват" и декодират с анимация (sci-fi ефект).

### 🌊 A4. Liquid Filter Transitions
При смяна на филтър, картите "плуват" до новата позиция с ефект на вода (spring physics + layout animation).

### ⚡ B. Калкулатор за Икономии на Ток
Потребителят въвежда стар климатик + цена на ток → вижда "Спестявате ≈ 420 лв/год. Изплаща се за 2.8 год." Живо обновяване, animated counter.

### 📐 C. Калкулатор на Мощност по Помещение
Въвеждаш м² + изложение → получаваш нужните BTU + автоматично филтрирани подходящи модели.

### 🔄 D. Layout Animation при Филтриране *(Задължително)*
Framer Motion `layout` + `AnimatePresence` – картите плавно се преподреждат при смяна на филтър (вместо грубо презареждане). Spring physics за усещане за "тежест".

### 👁️ E. "Наскоро Разгледани" Лента
Хоризонтална лента с последните 4-5 разгледани продукта от `localStorage`. Slide-in анимация.

### 📊 F. Ценова Хистограма (Airbnb стил)
Над ценовия слайдер – малка хистограма с разпределението на продуктите. Колоните потъмняват извън избрания диапазон.

### 📐 F2. Interactive Size Guide
Визуален гид в Quick View модала, който показва реалните размери на климатика спрямо човешка фигура (scale comparison).

### 🔄 F3. Before/After Slider (за проекти)
В ProjectsSection - слайдер, който показва "преди" и "след" монтажа с draggable handle.

### 📱 F4. Virtual "Try in Your Room"
Upload на снимка на стаята + overlay на климатика с регулируем размер и позиция.

### 🏷️ G. Активни Филтри като Chips *(Задължително)*
`[Daikin ✕]  [A+++ ✕]  [WiFi ✕]  [Изчисти всички]` – под search bar-а. Анимирано появяване/изчезване.

### ❤️ H. Любими + Споделяне *(Задължително)*
Иконка ❤️ (localStorage) + 🔗 копиране на URL. Тихи, елегантни, на всяка карта.

### ✨ I. Page Transition Анимация
При навигация главна → каталог: плавен fade + slide (Framer Motion AnimatePresence на router ниво).

### 🔍 J. Instant Search с Highlight

## 🗄️ 12. СТРАТЕГИЯ ЗА ДАННИТЕ (Frontend → Backend-ready)

**Принцип:** Всички данни живеят в един файл, изолирани от UI компонентите. Когато дойде backend, се заменя само функцията за зареждане — нищо друго не се пипа.

### Структура на data слоя:

```
components/
└── data/
    ├── catalog/
    │   ├── products.ts        ← Главният файл с всички продукти
    │   ├── categories.ts      ← Категории и метаданни
    │   ├── brands.ts          ← Марки (лого, описание, цвят)
    │   └── accessories.ts     ← Аксесоари и резервни части
    └── types/
        └── product.ts         ← TypeScript интерфейси
```

### TypeScript интерфейс (product.ts):
```typescript
export interface Product {
  id: string;
  brand: string;
  model: string;
  type: 'wall' | 'multi-split' | 'cassette' | 'accessory' | 'part';
  category: string;
  image: string;
  price: number;
  priceWithMount: number;
  rating: number;
  reviews: number;
  energyClass: string;
  badge?: { text: string; bg: string; textCol: string };
  specs: { icon: string; text: string }[];
  extras: string[];
  features: string[];         // За филтриране (WiFi, Инвертор и т.н.)
  power: number;              // BTU – за сортиране и филтриране
  inStock: boolean;
  cardBorder: string;
  imgBg: string;
  description?: string;       // За Quick View модала
}
```

### Изолиращ слой (service функции):
```typescript
// src/data/catalog/productService.ts
// ЗАМЕНИ САМО ТЕЗИ ФУНКЦИИ КОГАТО ИМАШ BACKEND:

export const getAllProducts = (): Product[] => {
  return products; // ← Замени с: return await fetch('/api/products')
};

export const getProductById = (id: string): Product | undefined => {
  return products.find(p => p.id === id); // ← Замени с API call
};

export const getProductsByCategory = (cat: string): Product[] => {
  return products.filter(p => p.category === cat);
};
```

> ⚠️ **Правило:** Компонентите НИКОГА не четат директно масива `products`. Те само извикват функциите от `productService.ts`. Така при смяна с реален API → промяна само в 1 файл!
